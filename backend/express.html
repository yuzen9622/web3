什麼是Express

Express 是一個用於 Node.js 的快速、不拘一格、極簡主義的Web框架。 Express 是一個最小且靈活的 Node.js Web 應用程式框架，它為 Web 和行動應用程式提供了一組強大的功能。

  • 替不同 HTTP Method、不同 URL 路徑的 requests 編寫不同的處理方法

  • 透過整合「畫面」的渲染引擎來達到插入資料到樣板中產生 response

  • 設定常見的 web 應用設定，例如：連線用的 port 和產生 response 的樣板位置

  • 在 request 的處理流程中增加額外的「中間層」進行處理

雖然 Express 本身非常簡單，但開發者們已經創造相容的中間層套件來解決大部份 web 開發的問題，這些套件能處理 cookies, sessions，登入，URL 參數，POST 資料，安全標頭等等，你能在Express Middleware中找到這些套件的列表（以及其他流行的第三方套件）

Express的基本路由

Express 提供了指定特定 HTTP 動詞（GET、POST、SET等）和 URL 模式（「路由」）呼叫什麼函數的方法，以及指定使用什麼模板（「檢視」）引擎以及模板檔案所在位置的方法，以及使用什麼模板來呈現響應。您可以使用 Express 中間件來新增對 cookie、會話和使用者、取得POST/GET參數等的支援。

傳統的資料驅動網站中，web 應用程式會等待來自瀏覽器(或其他客戶端)的 HTTP Request，接收到 Request 後根據 URL 和可能夾帶的POST/GET資料來決定需要回應什麼動作，根據需要可能對資料庫進行讀寫或執行滿足 Request 所需的其他任務。web 應用程式會回應 Response 給瀏覽器，通常是藉由插入檢所到的資料到 HTML 模板中動態產生 HTML 頁面讓瀏覽器顯示。

Express 的程式碼長怎樣?

JS

var express = require("express");
var app = express();

app.get("/", function (req, res) {
  res.send("Hello World!");
});

app.listen(3000, function () {
  console.log("Example app listening on port 3000!");
});

前兩行require()（導入）express 模組並建立Express 應用程式。該物件傳統上稱為app，具有用於路由 HTTP 請求、配置中間件、呈現 HTML 視圖、註冊模板引擎以及修改控制應用程式行為方式的應用程式設定（例如環境模式、路由定義是否區分大小寫、 ETC。 ）

程式碼的中間部分（以 開頭的三行app.get）顯示了路由定義。此方法指定一個回呼函數，只要存在具有相對於站點根目錄的路徑 ( ) 的app.get()HTTP 請求，就會呼叫該回呼函數。回調函數將請求和回應物件作為參數，並簡單地呼叫回應以傳回字串“Hello World!”GET'/'send()

最後一個區塊在連接埠「3000」上啟動伺服器並將日誌註釋列印到控制台。伺服器運行後，您可以localhost:3000在瀏覽器中查看傳回的範例回應。
