什麼是Spring Boot

Spring Boot 是目前 Java 後端中最主流的開發框架

幫助那些使用 Java 程式語言來撰寫後端程式的工程師，快速去架設一個後端系統起來的好用工具。 基本上 Spring Boot 就是 Java 後端中的一個好用的開發框架，只要你是使用 Java 來開發後端程式，那通常就是會使用 Spring Boot 這套框架，來加速你開發後端程式的效率

很多人都以為Spring boot是程式碼產生器，但是其實Spring boot並沒有產生code，且以為Spring boot是用來產生application server或產生web server，但是其實Spring boot是用來整合embedded servers(像是 tomcat、Jetty)。

那Spring boot主要是用來啟動專案，假設你現在要產生一個web server，需要有MVC架構、Spring core、validation framework…，你需要設定很多東西來達到這些需求，如果使用Spring boot起專案就會非常快速。

Spring boot提供以下幾種啟動專案類型：

  • Web

  • JPA

  • Enbedded Servers：假設想要啟動一個web application並且部署到linux box。在以往如果要這樣做需要安裝linux box，接著安裝Java，接著在安裝web server(Tomcat, weblogic , webspher)，接著再將application war 部署到這台機器。如果使用Spring boot，你可以打包你的server，像是你可以將你的tomcat 打包到你的application jar，這樣linux box就不需要在安裝web server只需要Java即可。

Spring Boot的優勢

優勢一：簡化 Spring 開發 使用 Spring Boot 的第一個好處，就是可以「簡化 Spring 的開發」

優勢二：快速整合主流框架 使用 Spring Boot 開發的第二個優勢，就是可以「快速的整合主流框架」

以下列出了 Spring Boot 與傳統 Java 框架之間的幾個主要差異：

  1. 簡化的配置：Spring Boot 採用許多預設值來配置應用程式，這些預設值可節省開發人員的時間和精力。傳統的 Java 框架通常需要在配置文件中進行大量的手動配置。

  2. 簡單的開發：Spring Boot 提供了豐富的快速開發工具，如 Spring Initializr，可以快速構建一個基本的 Spring Boot 應用程式，減少了開發人員的工作量。

  3. 微服務：Spring Boot 非常適合構建微服務架構。Spring Boot 可以使用 Spring Cloud 框架來實現服務發現、負載均衡和數據處理等功能。

  4. 開箱即用的功能：Spring Boot 預置了許多常用的功能，如資料庫連接、安全性、日誌紀錄和許多第三方庫的支持等。這些功能使得開發人員可以更快速地構建應用程式，而不必花費太多時間來處理底層細節。

使用 Spring Boot 開發 Web 應用程式還有其他的好處，例如：

  1. 自動配置：Spring Boot 可以自動配置您的應用程式，包括自動配置數據庫連接、自動配置 Web 應用程式、自動配置安全性等。

  2. 易於管理：Spring Boot 非常容易管理和維護。您可以使用一些內建的功能來管理應用程式，例如健康檢查、日誌管理、性能監控等。

  3. 優化的性能：Spring Boot 可以幫助您優化應用程式的性能，包括自動配置快取、使用內存數據庫等。

  4. 可擴展性強：Spring Boot 非常容易擴展，您可以很容易地添加新的功能和服務，並將它們整合到現有的應用程式中。

Spring Boot範例

@Required ： 它適用於 bean 設定方法。 它指示必須在設定時使用必需的屬性填充帶註解的Bean，否則它將引發異常 BeanInitilizationException。 

Java

public class Machine {
  private Integer cost;
  @Required
  public void setCost(Integer cost) {
  this.cost = cost;
  }
  public Integer getCost() {
  return cost;
  } 
}

@Autowired： Spring透過提供@Autowired注解來提供基於註解的自動裝配。 它用於自動連接setter方法，實例變數和構造函數上的spring bean。 當我們使用@Autowired批注時，spring容器通過匹配數據類型自動連接bean。 

Java
@Component
public class Customer{
private Person person;
@Autowired
public Customer(Person person){ 
this.person=person;
}
}

@Configuration ： 它是一個類級別的注解。 帶有@Configuration注解的類由Spring Containers用作bean定義的源。 

Java
@Configuration
public class Vehicle{
  @BeanVehicle engine(){
  return new Vehicle();
  }
}

@ComponentScan ： 當我們要掃描套件中的bean時使用。 它與註解@Configuration一起使用。 我們還可以指定用於掃描Spring元件的基本軟體包。 

Java
@ComponentScan(basePackages = "com.nhooo")
@Configuration
public class ScanComponent{
// ...
}

@Bean ： 是方法級的註解。 它是XML標記的代替方法。 它告訴產生由Spring Container管理的bean的方法。 

Java
@Bean
public BeanExample beanExample(){
return new BeanExample ();
}
